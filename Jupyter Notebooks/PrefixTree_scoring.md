# Скоринг для базового `PrefixTree`

Скоринг — это **средний минус-логарифм условной вероятности каждого токена при контексте предыдущих**. По сути это perplexity-подобная мера: чем больше значение, тем «удивительнее» сеанс с точки зрения модели.

## Формула

Для последовательности $x_1, x_2, \ldots, x_n$ скор считается так:

$$\text{NLL}(x) = \frac{1}{n} \sum_{i=1}^{n} -\log P(x_i \mid x_{i-k}, \ldots, x_{i-1})$$

где $k = \texttt{max\_depth} - 1$. То есть мы строим Марковскую модель порядка $k$ — каждый токен «зависит» только от $k$ предшествующих.

## Как считается сама вероятность $P(x_i \mid \text{ctx})$

Здесь и пригождается trie: в дереве узел `ctx` хранит счётчик `count`, равный числу раз, когда n-грам `ctx` встретился в обучении. Используется сглаживание Лапласа (add-$\alpha$):

$$P(x_i \mid \text{ctx}) = \frac{c(\text{ctx}, x_i) + \alpha}{c(\text{ctx}, *) + \alpha \cdot V}$$

Расшифровка:

- $c(\text{ctx}, x_i)$ — счётчик дочернего узла `ctx + (x_i,)`. В коде: `self.G.nodes[child]["count"]`.
- $c(\text{ctx}, *)$ — сумма счётчиков всех потомков узла `ctx`, то есть сколько раз контекст `ctx` встречался в принципе. В коде: `sum(self.G.nodes[c]["count"] for c in self.G.successors(node))`.
- $V$ — размер словаря (`len(self.vocab)`).
- $\alpha$ — параметр сглаживания (`smoothing`, в ноутбуке `0.3`).

## Зачем сглаживание

Без $\alpha$ незнакомому переходу присваивалось бы $P = 0$, а $-\log 0 = +\infty$ — одна неизвестная пара ломала бы весь скор. Add-$\alpha$ «размазывает» немного массы на все возможные продолжения: даже если данной комбинации $(\text{ctx}, x_i)$ мы не видели, у неё будет $P = \alpha / (c(\text{ctx},*) + \alpha V) > 0$.

## Что если сам контекст незнаком

Если узла `ctx` нет в дереве, мы не можем посчитать ни $c(\text{ctx}, x_i)$, ни $c(\text{ctx}, *)$, и в коде есть отдельная ветка:

```python
p = self.smoothing / (self.smoothing * V)   # = 1/V
```

То есть откатываемся на равномерное распределение по всему словарю.

## Почему среднее, а не сумма

Если суммировать $-\log P$, длинные сеансы автоматически получали бы больший «штраф» только за длину. Деление на $n$ нормализует: одинаково «удивительный» на каждом шаге сеанс получит одинаковый скор независимо от длины.

## Маленький пример

Допустим, `max_depth=3`, словарь $V=4$, $\alpha=0.5$, и в обучении мы видели бигрэмм $(4624, 4672)$ 8 раз и $(4624, 4625)$ 2 раза, других продолжений `4624` не было. Тогда у узла `(4624,)` два потомка с суммой $c(\text{ctx}, *) = 10$.

| Переход | Расчёт | $P$ | $-\log P$ |
|---------|--------|-----|-----------|
| $4624 \to 4672$ | $(8 + 0.5)/(10 + 0.5 \cdot 4) = 8.5/12$ | $\approx 0.708$ | $\approx 0.345$ |
| $4624 \to 4625$ | $(2 + 0.5)/12$                          | $\approx 0.208$ | $\approx 1.569$ |
| $4624 \to 4688$ | $(0 + 0.5)/12$                          | $\approx 0.042$ | $\approx 3.178$ |

Никогда не виденный переход штрафуется, но не бесконечно.

## Что конкретно код делает на каждом шаге $i$

1. Берёт контекст `ctx = seq[max(0, i - max_depth + 1):i]` — до $k$ предыдущих токенов (для первых позиций контекст укорачивается).
2. Ищет узел `ctx` в графе. Если его нет — `p = 1/V`.
3. Если есть — считает `ctx_total` (сумму счётчиков потомков), `tok_count` (счётчик нужного потомка) и применяет add-$\alpha$.
4. Накапливает $-\log p$ в `total`, инкрементирует счётчик `cnt`.
5. В конце возвращает `total / cnt`.

## Как интерпретировать результат

- $\text{NLL} \approx \log V$ (~$\log 6 \approx 1.79$ для словаря из шести токенов) — чисто случайная последовательность.
- У «нормальных» сеансов NLL низкая: большинство переходов высоковероятны, хорошо знакомы дереву.
- У атакующего сеанса с `mimikatz → ntdsutil` или `4625 → 4625 → 4625` контексты редкие или вовсе незнакомые — каждый такой шаг даёт большое $-\log p$, и средний NLL уезжает вверх.

Поэтому в ноутбуке мы сортируем сеансы по `nll desc` и смотрим на топ-кандидатов в подозрительные.

## Функция `anomalous_prefixes`

Использует ту же арифметику, но применяется *поточечно* — не усредняет, а возвращает список самых «дорогих» позиций в одном сеансе вместе с их контекстом. Это то, что в выводе выглядит как `4625 → 4625 → 4625 → 4624`: модель показывает не просто *что* подозрительно, а *где конкретно* в сеансе и *с каким контекстом*.
